AgentOS includes **25 pre-configured MCP (Model Context Protocol) integrations** that connect agents to external services via standardized stdio-based servers.

Integrations enable agents to interact with GitHub repos, Slack workspaces, Jira projects, databases, cloud APIs, and more — without custom tool code.

## Quick Start

```bash
# List available integrations
agentos integrations

# Add an integration
agentos add github --key $GITHUB_TOKEN

# Add with environment variables
export SLACK_BOT_TOKEN=xoxb-...
agentos add slack

# Test integration
agentos integration test github

# Remove integration
agentos remove github
```

## How MCP Works

MCP integrations run as **stdio-based child processes** that expose tools to agents:

```
┌─────────────┐
│   Agent     │
│  (Rust/TS)  │
└──────┬──────┘
       │
       │ MCP Protocol (JSON-RPC over stdio)
       │
┌──────▼──────────────────────────────────┐
│  @modelcontextprotocol/server-github    │
│  (Node.js subprocess)                   │
└──────┬──────────────────────────────────┘
       │
       │ GitHub REST API
       │
┌──────▼──────┐
│   GitHub    │
└─────────────┘
```

Agents send tool calls via JSON-RPC, the MCP server executes the operation, and returns results.

## Integration Configuration

Integrations are defined in TOML format:

```toml
[integration]
id = "github"
name = "GitHub"
description = "Access GitHub repositories, issues, pull requests, and actions"
category = "developer"
transport = "stdio"
command = "npx"
args = ["-y", "@modelcontextprotocol/server-github"]

[integration.env]
GITHUB_PERSONAL_ACCESS_TOKEN = { required = true, description = "GitHub personal access token with repo scope" }

[integration.oauth]
enabled = true
provider = "github"
scopes = ["repo", "read:org"]
```

<Accordion title="From configuration file">
  ```bash
  # Create custom integration
  cat > my-integration.toml <<EOF

  id = "custom-api"
  name = "Custom API"
  transport = "stdio"
  command = "node"

  agentos integration install my-integration.toml --key $API_KEY
  ```
</Accordion>

## Agent Integration Access

Control which integrations an agent can use:

```toml
[agent.capabilities]
# Grant access to all GitHub tools
tools = ["github::*"]

# Grant access to specific tools only
tools = [
  "github::github_create_issue",
  "github::github_list_repos"
]

# Grant access to multiple integrations
tools = [
  "github::*",
  "slack::*",
  "jira::*"
]
```

## Security Best Practices

<Card title="Credential Storage" icon="lock">
  All API keys and tokens are stored in the **encrypted vault** using AES-256-GCM with PBKDF2 key derivation.
</Card>

<Card title="Least Privilege" icon="shield">
  Grant agents access only to specific integration tools they need, not wildcard access.
</Card>

<Card title="Token Rotation" icon="rotate">
  Regularly rotate API tokens and update integrations using `agentos integration update`.
</Card>

<Card title="OAuth Preferred" icon="key">
  Use OAuth flows when available for automatic token refresh and scope management.
</Card>

<Card title="Environment Isolation" icon="layer-group">
  Use different credentials for development, staging, and production realms.
</Card>

## Creating Custom Integrations

Build custom MCP servers for proprietary APIs:

<CodeGroup>
```typescript custom-mcp-server.ts
import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';

const server = new Server(
  {
    name: 'custom-api-server',
    version: '1.0.0',
  },
  {
    capabilities: {
      tools: {},
    },
  }
);

// Register tools
server.setRequestHandler('tools/list', async () => ({
  tools: [
    {
      name: 'custom_api_call',
      description: 'Call custom API endpoint',
      inputSchema: {
        type: 'object',
        properties: {
          endpoint: { type: 'string' },
          method: { type: 'string', enum: ['GET', 'POST'] }
        },
        required: ['endpoint']
      }
    }
  ]
}));

server.setRequestHandler('tools/call', async (request) => {
  if (request.params.name === 'custom_api_call') {
    // Implement API call logic
    const result = await fetch(...);
    return { content: [{ type: 'text', text: JSON.stringify(result) }] };
  }
});

const transport = new StdioServerTransport();
await server.connect(transport);
```

```toml custom-api.toml
[integration]
id = "custom-api"
name = "Custom API"
description = "Integration with internal API"
category = "custom"
transport = "stdio"
command = "npx"
args = ["tsx", "./custom-mcp-server.ts"]

[integration.env]
CUSTOM_API_KEY = { required = true, description = "API key for custom service" }
CUSTOM_API_URL = { required = true, description = "Base URL for API" }
```

```bash Install
agentos integration install custom-api.toml \
  --key $CUSTOM_API_KEY \
  --env CUSTOM_API_URL=https://api.example.com
```
</CodeGroup>

## Troubleshooting

<Accordion title="Integration won't connect">
  ```bash
  # Check integration status
  agentos integration status github
  
  # View detailed logs
  agentos logs --filter integration=github --lines 50
  
  # Test with verbose output
  agentos integration test github --verbose
  ```
</Accordion>

<Accordion title="Missing tools in agent">
  ```bash
  # Verify agent has tool access
  agentos agent describe my-agent | grep capabilities
  
  # Grant integration access
  # Edit agent.toml:
  [agent.capabilities]
  tools = ["github::*"]
  
  # Reload agent
  agentos agent reload my-agent
  ```
</Accordion>

<Accordion title="Rate limit errors">
  Most integrations respect rate limits. Check provider documentation:
  
  - **GitHub**: 5,000 requests/hour (authenticated)
  - **Slack**: Tier-based limits
  - **Jira**: 5 requests/second per user
  
  Configure rate limiting in agent capabilities:
  ```toml
  [agent.resources]
  max_requests_per_hour = 1000
  ```
</Accordion>
